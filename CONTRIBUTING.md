# Contributing to Pit Stop 🏎️

First off, thank you for considering contributing to Pit Stop! It's people like you who make the F1 community and open-source world such an amazing place.

## How Can I Contribute?

### Reporting Bugs
- Check the issues tab to see if the bug has already been reported.
- If not, open a new issue. Clearly describe the problem and include steps to reproduce it.

### Suggesting Enhancements
- Open an issue with the tag "enhancement".
- Describe the feature you'd like to see and why it would be useful.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`python tests/check_system.py`).
5. Make sure your code lints (`npm run lint`).

## Style Guidelines

### Python
- Follow PEP 8.
- Use type hints wherever possible.

### JavaScript/TypeScript
- We use Prettier for formatting.
- Components should be modular and located in `components/`.
- Use TypeScript for all new logic.

## Questions?
Feel free to open an issue or reach out to the maintainers!
